[6] Remove Element

题目

  • 概述
    在数组中删除给的值。

  • 思路
    （错误思路）循环遍历，用后面的非val替换第一个val
    因为这样不能即使记录第一个val的位置。

（思路2）
将最后的非Val元素与第一个Val替换。
需要维护一个指待最后非val位置的指针。

另外需要注意的是
1.空数组要特判
2.返回的长度是下标+1，不是下标本身。
3.当前指针所指待位置是val时，下标要-1

  • 优秀代码
    思路相同，但这份代码通过将for改为while，减少了指针运用及查找非elem元素

[9] Valid Mountain Array

题目

  • 概述
    判断给的数组是否是前段严格递增，后段严格递减。（长度必须大于3）

  • 思路
    暴力。
    需要特别提防直接递减的序列。

  • 优秀代码
    思路相同，学习代码。
  1. 循环从1开始，相比从0开始，可以减去循环外再判断。
  2. 维持条件，节约了判断条件的编写。

[10] Replace Elements with Greatest Element on Right Side

题目

  • 概述
    将某一位替换为右侧的最大值，最后一位变成-1。

  • 思路
    1.暴力（纯暴力，也就是每次都搜，会超时。）
    2.感觉可以线段树，先存储每个区间最大值。

  • 优秀代码
    从右往左遍历就无需特意为最后一值赋-1，也无需回复遍历

[11] Move Zeroes

题目

  • 概述
    用最少的操作，将0元素移到最后，同时保留原有非0元素的顺序。

  • 思路
    用队列标0的位置，每次用非0值与队列首元素位置交换。

  • 优秀代码
    用一个count当了指针，把空间降到了O（1）

[14] Third Maximum Number

题目

  • 概述
    对于非空数组，找第三大的数，不存在就找最大值。时间复杂度要在O（n）
    （对于1，1，1，2序列，因为重复1被视为一个数，所以不存在第三大值）

  • 思路
    放到set里，自动从小到大排。注意判断长度是不是大于3。返回len-3

  • 优秀代码
    都使用set的默认排序（升序），但一旦超过3每次都删除最小值，保证了最后的访问效率。
    同时不是什么都插入，只有大于最小值的才插入。

[15] Find All Numbers Disappeared in an Array

题目

  • 概述
    对于[1,n]，找缺失的数。不要额外空间，时间复杂度O（N）

  • 思路
    遍历，将所有数，对应的原位置的数，变为相反数（不能单纯赋特定值，否则失去后面位置的指待）。则最后剩下的正数位置i+1为缺失值。
