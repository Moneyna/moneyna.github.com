5 遇到的问题

  • 后端没有处理前端请求
    上面提到过的路径设定问题。在内部project的urls.py引入application的urls.py时，可能会写错路径。这个时候是改前端路径，还是改后端的逻辑就看个人选择了。

  • 整合后的工程运行后，前端运行不正常
    跟其他所有组件开发的工程一样，出错的原因无外乎三种，一是前端本身有错，而是后端有错，三是两个整合后出错。
    一般是前端本身有问题，debug过程最好是直接去debug前端，不要去调试npm run build后的文件（这时候报错内容已经与开发时不同了）。
